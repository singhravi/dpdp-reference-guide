Key Concepts & Obligations

Critical definitions, roles, and government-specific applicability under DPDP Act 2023.

Government ContextThe DPDP Act applies to Govt Departments processing personal data of citizens in digital form. Section 17 provides specific exemptions for State instrumentalities. Departments should map each service to determine applicability before starting compliance work.
🏛️

Data Fiduciary (Your Dept.)

Section 2(i) — DPDPA 2023

Any Govt Department that determines the purpose and means of processing personal data of citizens. Includes Ministries, State Depts, PSUs, Autonomous Bodies.

  • Collect only for specified, explicit purpose
  • Provide privacy notice before or at collection
  • Ensure accuracy and completeness of data
  • Implement appropriate security safeguards
  • Respond to citizen rights requests within 72 hrs
  • Report breaches to Data Protection Board
👤

Data Principal (Citizen)

Section 2(j) — DPDPA 2023

The individual (citizen) to whom personal data relates. Has statutory rights the Government portal must honour. In case of a minor, the parent/guardian is the Data Principal.

  • Right to access information about data processing
  • Right to correction and erasure
  • Right to grievance redressal
  • Right to nominate a nominee
  • Consent can be withdrawn at any time
  • Right to know if data shared with third parties
📋

Personal Data — What's Covered

Section 2(t) — DPDPA 2023

Any data about an identifiable individual. Government portals routinely handle vast amounts across citizen services.

  • Name, address, phone number, email
  • Aadhaar, PAN, Voter ID, Passport numbers
  • Biometric data collected during service delivery
  • Health, income, caste certificate data
  • IP addresses, device IDs, usage logs
  • Bank account & financial details
⚠️

Penalties for Non-Compliance

Section 33 — DPDPA 2023

Data Protection Board can levy significant penalties. Government Departments are not fully exempt.

  • Failure of security safeguards: up to ₹250 Cr
  • Failure to notify breach: up to ₹200 Cr
  • Violation of children's data provisions: up to ₹200 Cr
  • Non-compliance with DPB directions: up to ₹150 Cr
  • Breach of other obligations: up to ₹50 Cr

3-Phase Implementation Roadmap

Achieve full DPDP compliance in 16 weeks — Phase 1 requires zero code change; Phases 2 & 3 involve minimal configuration-level work.

Zero-Code PrincipleMost DPDP obligations can be met through policy documents, consent overlays injected via tag managers, and separate privacy dashboards — without modifying the core application codebase.
1

Phase 1 — Assess, Document & Policy (Weeks 1–4)

⏱ 0 code changes required · All departments must start here
  • Conduct Data Mapping: list all forms, APIs, databases that collect personal data
  • Classify data categories: general, sensitive, children's data
  • Identify existing legal bases: statutory obligation, public function, or consent
  • Draft / update Privacy Policy document (standalone page on portal)
  • Draft Notice Template for each data collection form (can be PDF/static page)
  • Appoint a Data Protection Officer (DPO) or Grievance Redressal Officer
  • Create Data Retention & Deletion Schedule per data category
  • Identify all third-party service providers and review agreements
  • Register with the Data Protection Board when registration portal is live
2

Phase 2 — Consent Layer & Notice Injection (Weeks 5–10)

⚙️ Configuration-level only — tag manager or reverse proxy injection
  • Deploy a Consent Management Platform (CMP) via Google Tag Manager or NIC's solution
  • Inject consent banner/overlay on entry to all service portals — no backend change needed
  • Link "Privacy Notice" to each form's submission button via JavaScript overlay
  • Implement consent withdrawal link in portal header (links to separate consent dashboard)
  • Add cookie consent banner for analytics/tracking cookies (configurable via CMP)
  • Enable multilingual notice display using existing i18n framework or CMP feature
  • Set up separate Grievance Redressal web form (can be a standalone microsite)
  • Add "Request my data / Delete my data" links in user account section
3

Phase 3 — Rights Fulfilment & Audit (Weeks 11–16)

🔒 Minimal code — separate privacy microsite handles citizen requests
  • Build standalone Privacy Dashboard (a separate application, not touching core app)
  • Implement right to access: generate data summary report for citizen on request
  • Implement right to correction: internal workflow to update records on citizen request
  • Implement right to erasure: define erasure scope (exclude mandatory statutory records)
  • Implement nomination feature: form for citizen to nominate data representative
  • Create Data Breach Response SOP and test notification mechanism
  • Conduct first annual security audit (CERT-In empanelled auditor)
  • Staff training for DPO, grievance officers, and front-line teams
🛡️

No-Code Tools Allowed

Achieve compliance without touching core app
  • Google Tag Manager / NIC Tag Manager for consent banners
  • Nginx/Apache reverse proxy for injecting privacy headers & notices
  • Standalone microsite for Privacy Dashboard (separate domain)
  • ServiceNow / CRM for Grievance Redressal workflow
  • PDF consent notice linked from existing forms
📄

Legal Basis: Statutory Function

Key advantage for Govt Departments

Govt Departments performing statutory / sovereign functions may rely on Section 7(c) rather than consent for mandatory data collection (e.g., tax filing, ration card, pension).

  • Map each service to its enabling statute
  • Notice still required — only consent mechanism differs
  • Cannot process beyond the statutory purpose

    Consent Notice — Rules 2025 Requirements

    Rule 3 — Mandatory elements in notice
    • Name & contact of Data Fiduciary (Dept name, address, DPO email)
    • Purpose of each data element being collected
    • Categories of personal data collected
    • Whether data will be shared & with whom
    • Period for which data will be retained
    • How citizen can exercise their rights
    • Link to full privacy policy
    • Available in all 22 scheduled languages
    ⚡

    Consent is NOT Required For

    Section 7 exemptions — Govt specific

    Government Departments can rely on legitimate bases other than consent for mandatory citizen services:

    • State function under law: Tax filing, PAN, pension, election ID
    • Medical emergency: Health data during emergencies
    • Disaster/public order: Data for disaster relief operations
    • Employment / payroll: Govt employee data for salary, benefits
    • Judicial proceedings: Data required by courts or tribunals
    Notice is still mandatory even when consent is not required.

    8 Rights of Citizens (Data Principals)

    Sections 11–16 of DPDP Act 2023. Government Departments must provide mechanisms to fulfil each right — most can be implemented as a separate grievance/request portal.

    📂

    Right to Access Information

    Section 11 — DPDPA 2023

    Citizen can request a summary of personal data being processed, the purposes, and list of Data Processors/entities with whom data is shared.

    • Respond within 30 days of request
    • Provide data in readable format (PDF / email)
    • No fee chargeable (first request)
    • Implementation: Standalone request form + manual/semi-automated report
    Separate portal 30-day SLA
    ✏️

    Right to Correction & Erasure

    Section 12 — DPDPA 2023

    Citizen can demand correction of inaccurate, incomplete, or outdated data; or erasure when data is no longer needed or consent is withdrawn.

    • Government can refuse erasure where retention is mandatory under law
    • Correction workflow: DPO reviews → updates → informs citizen
    • Implementation: Grievance form + internal ticketing system
    • Must inform other Data Processors of the correction
    High priority
    🚫

    Right to Withdraw Consent

    Section 13 — DPDPA 2023

    Citizen can withdraw consent given earlier. Withdrawal does not affect legality of processing done before withdrawal.

    • Must be as easy as giving consent
    • Process within 30 days of withdrawal
    • Department may stop optional services post-withdrawal
    • Implementation: Link in portal header → Consent Dashboard microsite
    Critical
    🛡️

    Right to Grievance Redressal

    Section 14 — DPDPA 2023

    Citizen can file a grievance against any violation of their rights under the Act. Rules mandate 48-hour acknowledgement and 30-day resolution.

    • Appoint Grievance Redressal Officer (GRO) — must be Indian resident
    • Publish GRO name, email, address on portal
    • Acknowledge within 48 hours, resolve within 30 days
    • Implementation: Separate web form or CPGRAMS integration
    Critical
    ⚖️

    Right to Appeal to DPB

    Section 15 — DPDPA 2023

    If grievance is not resolved satisfactorily, citizen can appeal to the Data Protection Board of India.

    • Citizen must first exhaust grievance with the Department
    • Departments must respond to DPB notices within prescribed time
    • Implementation: Ensure grievance records are archived for DPB inspection
    High
    👤

    Right to Nominate

    Section 16 — DPDPA 2023

    Citizen can nominate another individual to exercise their data rights on their behalf in case of death or incapacity.

    • Nomination can be done at time of consent or anytime later
    • Nominee must be verified and authenticated
    • Relevant particularly for elderly/differently-abled citizens
    • Implementation: Optional field in privacy dashboard

    Government Exemptions & Special Clauses

    Section 17 of DPDP Act 2023 — Specific provisions that apply to Central and State Government entities.

    ImportantExemptions are narrow and must be invoked correctly. Blanket reliance on exemptions without documentation exposes Departments to regulatory risk. Each exemption invocation must be justified and recorded.
    🔐

    National Security Exemption

    Section 17(1)(a) — DPDPA 2023

    Central Govt may exempt any instrumentality in the interest of sovereignty, integrity, security, friendly foreign relations, or public order.

    • Must be notified by Central Government via official order
    • Cannot be self-invoked by individual departments
    • Applies to intelligence agencies, defence, border security
    Central Govt Only
    🔬

    Research & Statistics Exemption

    Section 17(2)(a) — DPDPA 2023

    Data processed for research, archiving, or statistical purposes is exempt from certain obligations provided appropriate safeguards are in place.

    • Must be anonymised or pseudonymised wherever possible
    • Cannot identify or profile specific citizens from research output
    • Applicable to: Census, NSSO surveys, ICMR health research
    Conditional
    ⚖️

    Judicial / Legal Proceedings

    Section 17(2)(b) — DPDPA 2023

    Processing of personal data for enforcement of legal rights, claims, or in connection with judicial or quasi-judicial proceedings is exempt from consent requirements.

    • Applies to: Courts, Tribunals, Law Enforcement, CBI, ED
    • Must be proportional to the purpose
    • Data must not be retained beyond proceeding completion
    With conditions
    📢

    Publicly Available Data

    Section 3(c) — DPDPA 2023

    Personal data that has been made public by the Data Principal themselves does not require fresh consent for processing.

    • The data principal must have voluntarily made it public
    • Government-published data (gazette) qualifies
    • Cannot be repurposed beyond original publication scope
    Narrow scope
    Best Practice for Govt DepartmentsEven where exemptions apply, maintain Privacy Notices, appoint a DPO, implement security safeguards, and create a grievance mechanism. This builds citizen trust and protects the Department in case of regulatory scrutiny.

    1About This Policy

    This Privacy Policy explains how the Ministry of Electronics and Information Technology (MeitY), Government of India ("we", "the Department") collects, uses, shares, and retains personal data of users ("citizens", "Data Principals") who access this DPDP Compliance Portal, in accordance with the Digital Personal Data Protection Act, 2023 and DPDP Rules, 2025.

    2What Personal Data We Collect

    • Name, designation, department name and official email address
    • Phone number (for OTP-based login or grievance contact)
    • Device information: IP address, browser type, operating system
    • Usage data: pages visited, time spent, checklist completion status (stored locally)
    • Form submissions: grievance forms and DPO contact queries

    3Purpose of Data Collection

    Data Element Purpose Legal Basis
    Name & Email User authentication, grievance response, DPO correspondence Statutory function / Consent
    Department Name Tailored compliance guidance and reporting Statutory function
    IP Address Security monitoring, fraud prevention Legitimate interest
    Usage Logs Portal performance analytics (anonymised) Consent via cookie banner
    Grievance Form Data Processing and resolving data protection grievances Statutory function

    4Data Sharing & Third Parties

    We do not sell, rent, or share your personal data with commercial third parties. Data may be shared with:

    • National Informatics Centre (NIC) — hosting and infrastructure
    • Data Protection Board of India — in case of formal complaint or inquiry
    • CERT-In — for cybersecurity incident response
    • Other Central Government Departments — only where legally required

    5Data Retention

    • Authentication logs: 90 days
    • Grievance records: 5 years (as per Public Records Act)
    • Usage analytics (anonymised): 24 months
    • DPO correspondence: 3 years
    • All data deleted or anonymised after retention period expires

    6Your Rights as a Data Principal

    Under the DPDP Act 2023, you have the right to: (a) access personal data held about you; (b) correct inaccurate or outdated data; (c) request erasure of data no longer required; (d) withdraw consent at any time; (e) file a grievance with the DPO; and (f) appeal to the Data Protection Board if your grievance is not resolved.

    7Security Measures

    We implement appropriate technical and organisational measures including SSL/TLS encryption, role-based access controls, annual CERT-In empanelled security audits, data minimisation practices, and incident response procedures aligned with DPDP Rules 2025 (Rule 6 & 7).

    8Cookies

    This portal uses: (a) Essential cookies — required for session management and security, exempt from consent; (b) Analytics cookies — used to understand portal usage, only with your explicit consent via the cookie banner. You may withdraw cookie consent at any time via the cookie settings link in the footer.

    Data Protection Board of India

    ⚖️

    Data Protection Board of India (DPBI)

    Statutory adjudicatory body established under Section 18 of the Digital Personal Data Protection Act, 2023. The Board adjudicates complaints, imposes penalties, and issues binding directions to Data Fiduciaries — including Government Departments.

    🏢 Establishment

    • Constituted by the Central Government
    • Chairperson + Members appointed by Govt
    • Operates as a digital-first adjudicatory body
    • All proceedings conducted digitally
    • Orders appealable to Telecom Disputes Settlement & Appellate Tribunal (TDSAT)

    ⚖️ Powers of the Board

    • Investigate complaints filed by Data Principals
    • Suo motu inquiry on receipt of information of breach
    • Issue binding directions to Data Fiduciaries
    • Impose financial penalties (up to ₹250 Crore)
    • Call for information, records, and documents
    • Refer matters to CERT-In for technical analysis

    How Citizens Approach the DPB

    • 1
      File Grievance with Data FiduciaryCitizen must first raise the issue with the Department's Grievance Officer. GRO must acknowledge within 48 hrs and resolve within 30 days.
    • 2
      Unsatisfied? Appeal to DPBIf the grievance is unresolved or citizen is unsatisfied with the outcome, they can file a complaint with the DPB via its digital portal.
    • 3
      DPB Issues Notice to DepartmentDPB issues a notice to the Data Fiduciary (your Department). The Department must respond within the time stipulated by the DPB.
    • 4
      DPB Hearing & OrderDPB conducts a digital hearing, examines evidence, and passes a binding order — which may include penalty imposition and/or directions to remedy the situation.
    • 5
      Appeal to TDSATEither party can appeal the DPB order to the Telecom Disputes Settlement & Appellate Tribunal (TDSAT) within 60 days of the order.
