Key Concepts & Obligations

Critical definitions, roles, and government-specific applicability under DPDP Act 2023 & Rules 2025.

Government ContextThe DPDP Act applies to Govt Departments processing personal data of citizens in digital form. Section 17 provides specific exemptions for State instrumentalities. Departments should map each service to determine applicability before starting compliance work.
🏛️

Data Fiduciary (Your Dept.)

Section 2(i) — DPDPA 2023

Any Govt Department determining purpose & means of processing personal data. Includes Ministries, State Depts, PSUs, Autonomous Bodies.

  • Collect only for specified, explicit purpose
  • Provide privacy notice before or at collection
  • Ensure accuracy & completeness of data
  • Implement appropriate security safeguards
  • Respond to citizen rights requests within 30 days
  • Report breaches to Data Protection Board within 72 hrs
👤

Data Principal (Citizen)

Section 2(j) — DPDPA 2023

The individual to whom personal data relates. Has 8 statutory rights the Government portal must honour. In case of a minor, the parent/guardian is the Data Principal.

  • Right to access information about data processing
  • Right to correction and erasure
  • Right to grievance redressal
  • Right to nominate a nominee
  • Consent can be withdrawn at any time
  • Right to appeal to Data Protection Board
📋

Personal Data — What's Covered

Section 2(t) — DPDPA 2023

Any data about an identifiable individual. Government portals routinely handle vast amounts across citizen services.

  • Name, address, phone number, email
  • Aadhaar, PAN, Voter ID, Passport numbers
  • Biometric data collected during service delivery
  • Health, income, caste certificate data
  • IP addresses, device IDs, usage logs
  • Bank account & financial details
⚠️

Penalties for Non-Compliance

Section 33 — DPDPA 2023

Data Protection Board can levy significant penalties. Government Departments are not fully exempt.

  • Failure of security safeguards: up to ₹250 Crore
  • Failure to notify breach: up to ₹200 Crore
  • Violation of children's data provisions: up to ₹200 Crore
  • Non-compliance with DPB directions: up to ₹150 Crore
  • Breach of other obligations: up to ₹50 Crore

3-Phase Implementation Roadmap

Full DPDP compliance in 16 weeks. Phase 1 = zero code changes. Phases 2 & 3 = configuration-level only.

Zero-Code PrincipleMost DPDP obligations can be met through policy documents, consent overlays injected via tag managers, and separate privacy microsites — without modifying the core application codebase.
1

Phase 1 — Assess, Document & Policy (Weeks 1–4)

⏱ 0 code changes required · All departments must start here
  • Conduct Data Mapping: list all forms, APIs, databases that collect personal data
  • Classify data categories: general, sensitive, children's data
  • Identify legal bases for each service: statutory obligation, public function, or consent
  • Draft / update Privacy Policy document (standalone static page on portal)
  • Draft Purpose-wise Notice Template for each data collection form (PDF or static HTML)
  • Appoint a Data Protection Officer (DPO) / Grievance Redressal Officer via official order
  • Create Data Retention & Deletion Schedule aligned with enabling statutes
  • Identify all third-party service providers (Data Processors) and review agreements
  • Register with the Data Protection Board when the DPB registration portal is live
2

Phase 2 — Consent Layer & Notice Injection (Weeks 5–10)

⚙️ Configuration-level only — tag manager or reverse proxy. No backend changes.
  • Deploy Consent Management Platform (CMP) via Google Tag Manager or NIC's approved solution
  • Inject consent banner/overlay on entry to all service portals — no backend change needed
  • Link "Privacy Notice" to each form's submission button via JavaScript overlay (GTM)
  • Implement consent withdrawal link in portal header linking to standalone consent dashboard
  • Add cookie consent banner for analytics/tracking cookies (configurable via CMP)
  • Enable multilingual notice display using existing i18n framework or CMP i18n feature
  • Set up Grievance Redressal web form (can be a standalone microsite or CPGRAMS)
  • Add "Request my data / Delete my data" links in user account section or footer
  • Configure HTTP security headers: HSTS, CSP, X-Frame-Options via Nginx/Apache
3

Phase 3 — Rights Fulfilment & Audit (Weeks 11–16)

🔒 Minimal code — separate privacy microsite handles all citizen requests
  • Build standalone Privacy Dashboard (separate application, not touching core app)
  • Implement Right to Access: generate data summary report for citizen on request
  • Implement Right to Correction: internal DPO workflow to update records on citizen request
  • Implement Right to Erasure: define scope (exclude mandatory statutory records)
  • Implement Nomination Feature: online form for citizen to nominate data representative
  • Activate and test Data Breach Response SOP and 72-hr DPB notification mechanism
  • Conduct first Annual Security Audit with CERT-In empanelled auditor
  • Review and update all Data Processor agreements to include DPDP obligations
  • Complete staff training for DPO, grievance officers, and citizen-service front-line teams
🛡️

No-Code Approved Tools

Achieve compliance without touching core app
  • Google Tag Manager / NIC Tag Manager for consent banners
  • Nginx/Apache reverse proxy for injecting privacy headers & notices
  • Standalone microsite for Privacy Dashboard (separate subdomain)
  • ServiceNow / CPGRAMS for Grievance Redressal workflow

Legal Basis: Statutory Function

Key advantage for Govt Departments — Section 7(c)

Govt Departments performing statutory / sovereign functions may rely on Section 7(c) — processing necessary for performance of any function under law — rather than consent for mandatory citizen services.

  • Map each service to its enabling statute
  • Notice is still mandatory even without consent requirement
  • Cannot process data beyond the statutory purpose

    When Consent is NOT Required

    Section 7 exemptions — Government specific

    Government Departments can rely on legitimate bases other than consent for mandatory citizen services:

    • State function under law: Tax filing, PAN, pension, election ID
    • Medical emergency: Health data during emergencies by govt hospitals
    • Disaster/public order: Data for disaster relief operations
    • Employment / payroll: Govt employee data for salary, benefits
    • Judicial proceedings: Data required by courts or tribunals
    Notice is still mandatory even when consent is not required.

    8 Rights of Citizens (Data Principals)

    Sections 11–16 of DPDP Act 2023. Departments must provide mechanisms to fulfil each right — most can be handled via a separate privacy microsite.

    📂

    Right to Access Information

    Section 11 · SLA: 30 days

    Citizen can request a summary of personal data processed, the purposes, and list of Data Processors with whom data is shared.

    • Respond within 30 days of request
    • Provide data in readable format (PDF / email)
    • No fee chargeable for first request
    • Impl: Standalone request form + DPO manual processing
    Separate portal
    ✏️

    Right to Correction & Erasure

    Section 12 · SLA: 30 days

    Citizen can demand correction of inaccurate/incomplete data; or erasure when data is no longer required or consent is withdrawn.

    • Government can refuse erasure where retention is mandatory under law
    • Correction workflow: DPO reviews → updates → informs citizen
    • Must inform downstream Data Processors of the correction
    • Impl: Grievance form + internal ticketing + DB update workflow
    High priority
    🚫

    Right to Withdraw Consent

    Section 13 · SLA: 30 days

    Citizen can withdraw consent given earlier. Withdrawal does not affect legality of processing done before withdrawal.

    • Must be as easy as giving consent — one-click or one-form
    • Process withdrawal within 30 days
    • Department may stop optional services post-withdrawal
    • Impl: Consent withdrawal link in portal header → Consent Manager microsite
    Critical
    🛡️

    Right to Grievance Redressal

    Section 14 · 48-hr ack · 30-day resolve

    Citizen can file a grievance against any violation of rights under the Act. Rules mandate 48-hour acknowledgement and 30-day resolution.

    • Appoint Grievance Redressal Officer (GRO) — must be Indian resident
    • Publish GRO name, email, address on portal prominently
    • Acknowledge within 48 hours, resolve within 30 days
    • Impl: Separate web form or CPGRAMS integration
    Critical
    ⚖️

    Right to Appeal to DPB

    Section 15 · As directed by DPB

    If grievance is not resolved satisfactorily, citizen can appeal to the Data Protection Board. DPB can direct the fiduciary to remedy.

    • Citizen must first exhaust departmental grievance process
    • Departments must respond to DPB notices within prescribed timelines
    • Impl: Archive all grievance records; designate DPB liaison officer
    High
    👶

    Minor Data / Parental Consent

    Section 9 & Rule 10 · Before collection

    Processing of personal data of a child (under 18) requires verifiable parental/guardian consent before any data is collected.

    • Age verification method prescribed in Rule 10
    • No tracking, profiling, or targeted advertising for minors
    • School/education portals especially affected
    • Impl: Age gate on registration page + parental consent OTP/form
    Critical
    👤

    Right to Nominate

    Section 16 · At request of citizen

    Citizen can nominate another individual to exercise their data rights on their behalf in case of death or incapacity.

    • Nomination can be done at time of consent or anytime later
    • Nominee must be verified and authenticated
    • Particularly relevant for elderly/differently-abled citizens
    • Impl: Simple form in privacy dashboard; DigiLocker integration optional
    Phase 3
    📊

    Data Minimisation & Purpose Limitation

    Sections 4 & 6 · Ongoing obligation

    Departments may only collect data that is necessary for the stated purpose and may not use it for any other purpose without fresh consent or lawful basis.

    • Audit all forms: remove non-essential fields
    • Cannot use census data for law enforcement without separate legal basis
    • Data shared with other departments needs its own lawful basis
    • Impl: Form audit in Phase 1; data flow diagrams required
    Ongoing

    Data Breach Response Protocol

    Section 8(6) & Rule 7 — Notify the Data Protection Board within 72 hours; notify affected citizens within 7 days.

    Zero Tolerance — 72-Hour Clock Starts from AwarenessThe moment your department becomes aware of a breach, the 72-hour clock starts ticking. Prepare and test your SOP now — not after an incident.
    ⏰

    Critical Timelines

    Rule 7 — DPDP Rules 2025
    • Immediate: Detect & confirm breach, isolate systems
    • 2 hours: Contain breach, revoke compromised credentials
    • 12 hours: Severity assessment, preliminary report to DPO
    • 72 hours: Mandatory notification to Data Protection Board
    • 7 days: Notify all affected Data Principals
    • 30 days: Technical remediation complete
    • 60 days: Post-incident review, file final report with DPB
    📋

    DPB Notification Must Include

    Rule 7(1) — Mandatory content
    • Nature and category of personal data compromised
    • Approximate number of Data Principals affected
    • Likely consequences of the breach
    • Measures taken or proposed to address the breach
    • Contact details of Data Protection Officer
    • Whether breach has been fully contained or is ongoing
    • Any technical forensic findings (preliminary)
    🔒

    Preventive Safeguards

    Rule 6 — Security standards
    • Annual CERT-In empanelled security audit (mandatory)
    • Implement HTTPS/TLS 1.3 on all citizen-facing portals
    • Role-based access control (RBAC) for all staff accessing personal data
    • Encrypted backups with tested restoration procedures
    • Multi-factor authentication for admin/backend access
    • Network segmentation: isolate databases from public-facing servers

    7-Step Breach Response Workflow

    • Step 1 — Detect & Confirm

      Immediate

      Identify the breach through SIEM alerts, user reports, or external notification. Confirm it is a genuine breach of personal data, not a false positive. Document time of detection and initial scope.

    • Step 2 — Contain & Isolate

      Within 2 hours

      Isolate affected systems from the network. Revoke all compromised credentials immediately. Preserve system logs and evidence for forensic analysis. Do not wipe or overwrite affected systems.

    • Step 3 — Assess Severity

      Within 12 hours

      Determine what data was compromised (categories, volume). Assess risk level: High (Aadhaar/health/financial), Medium (contact details), Low (anonymised). Document findings in the Incident Register.

    • Step 4 — Notify Data Protection Board

      Within 72 hours

      Submit formal breach notification to DPB via prescribed format (Rule 7). Include: nature of breach, data categories, number affected, likely consequences, containment measures, and DPO contact. If full details unavailable in 72 hrs, submit initial notice and follow up.

    • Step 5 — Notify Affected Citizens

      Within 7 days

      Inform all affected Data Principals in plain language (available in local language/Hindi). Describe: what data was exposed, potential risks, steps they can take to protect themselves, and how to reach the DPO/grievance officer.

    • Step 6 — Remediate

      Within 30 days

      Implement all technical fixes: patch vulnerabilities, update access controls, reset all potentially compromised credentials. Engage CERT-In empanelled security firm for independent assessment if required.

    • Step 7 — Post-Incident Review

      Within 60 days

      Conduct root cause analysis. Update security controls and breach SOP based on lessons learned. File final closure report with DPB. Update Breach Register. Brief senior management and DPO on outcomes.

    Build a Breach Register NowMaintain a running Breach Register documenting every security incident (even those that did not result in data exposure). This is essential for DPB compliance audits and demonstrates a culture of accountability.

    Government Exemptions & Special Clauses

    Section 17 of DPDP Act 2023 — specific provisions for Central and State Government entities. Exemptions are narrow and must be invoked carefully.

    Important — Exemptions Are Not BlanketEach exemption invocation must be justified, documented, and proportionate. Blanket reliance on exemptions without documentation exposes Departments to significant regulatory risk from the Data Protection Board.
    🔐

    National Security Exemption

    Section 17(1)(a) — DPDPA 2023

    Central Govt may exempt any instrumentality in the interest of sovereignty, integrity, security, friendly foreign relations, or public order.

    • Must be notified by Central Government via official gazette notification
    • Cannot be self-invoked by individual departments
    • Subject to procedural safeguards to be prescribed
    • Applies to: IB, RAW, NTRO, BSF, NSG, MHA security wing
    Central Govt Only
    🔬

    Research & Statistics Exemption

    Section 17(2)(a) — DPDPA 2023

    Data processed for research, archiving, or statistical purposes is exempt from certain obligations provided appropriate safeguards are in place.

    • Data must be anonymised or pseudonymised wherever possible
    • Cannot identify or profile specific individuals from research output
    • Publication must not contain personally identifiable information
    • Applies to: Census, NSSO surveys, ICMR health research, NITI Aayog
    Conditional
    ⚖️

    Judicial / Legal Proceedings

    Section 17(2)(b) — DPDPA 2023

    Processing for enforcement of legal rights, claims, or in connection with judicial or quasi-judicial proceedings is exempt from consent requirements.

    • Applies to: Courts, Tribunals, CBI, ED, NIA, Income Tax Department
    • Must be proportional to the purpose of the proceedings
    • Data must not be retained beyond proceeding completion
    • Notice may still be required post-proceedings where feasible
    With conditions
    📢

    Publicly Available Data

    Section 3(c) — DPDPA 2023

    Personal data that has been made public by the Data Principal themselves does not require fresh consent for processing.

    • The data principal must have voluntarily made it public
    • Government-published data (gazette, RTI responses) qualifies
    • Cannot be repurposed beyond the original scope of publication
    • Does not apply to leaked or unauthorised disclosures
    Narrow scope
    Best Practice for All Government DepartmentsEven where exemptions apply, maintain Privacy Notices, appoint a DPO, implement security safeguards, and provide a grievance mechanism. This builds citizen trust and protects the Department in case of DPB scrutiny. Exemption from the Act's provisions does not mean exemption from all data ethics obligations.

    Data Protection Board of India (DPBI)

    Statutory adjudicatory body established under Section 18 of the DPDP Act 2023. Adjudicates complaints, imposes penalties, and issues binding directions to Data Fiduciaries — including Government Departments.

    🏢 Establishment & Structure

    • Constituted by the Central Government
    • Chairperson + Members appointed by Govt
    • Operates as a fully digital adjudicatory body
    • All proceedings conducted online/digitally
    • Orders appealable to TDSAT within 60 days

    ⚖️ Powers of the Board

    • Investigate complaints from Data Principals
    • Suo motu inquiry on breach information
    • Issue binding directions to Data Fiduciaries
    • Impose penalties up to ₹250 Crore
    • Call for records, documents, and information
    • Refer technical matters to CERT-In

    How Citizens Approach the DPB

    • 1
      File Grievance with Data Fiduciary (Your Dept.)Citizen raises issue with Grievance Officer. GRO must acknowledge in 48 hrs, resolve in 30 days.
    • 2
      Unsatisfied → Appeal to DPBIf unresolved or citizen is unsatisfied, they file a complaint with the DPB via its digital portal.
    • 3
      DPB Issues Notice to DepartmentDPB issues notice to your Department. Must respond within the time stipulated by the DPB.
    • 4
      DPB Hearing & Binding OrderDPB conducts digital hearing, examines evidence, passes binding order — may include penalty.
    • 5
      Appeal to TDSATEither party can appeal DPB order to the Telecom Disputes Settlement & Appellate Tribunal within 60 days.
